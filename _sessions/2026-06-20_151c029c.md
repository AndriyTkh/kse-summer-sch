# Air Raid Alert Time Series — Problem Map

## What we can solve (problem framings)

**1. Forecasting** — predict next alert. "Will region X have alert in next N hours?" or "expected alerts tomorrow."
- Classification: alert / no-alert per time window.
- Regression: count of alerts per day/region.

**2. Duration prediction** — given alert started, how long lasts? Helps people decide shelter vs wait.

**3. Anomaly detection** — spot unusual spikes (mass attack days, new campaign phase). Flag deviation from baseline.

**4. Pattern/seasonality mining** — when do alerts cluster? Hour-of-day, day-of-week, lunar/weather, correlation with front-line events.

**5. Spatial-temporal spread** — alert in region A → predict propagation to B (drone/missile flight paths). Sequence/graph problem.

**6. Clustering regions** — group oblasts by alert behavior similarity.

---

## Data parameters

Core fields per alert event:
- `region` / oblast / hromada (spatial granularity)
- `start_timestamp`, `end_timestamp` → derive `duration`
- `alert_type` (air, artillery, chemical, etc.)
- derived time features: hour, weekday, month, is_weekend, days_since_invasion

Engineered:
- rolling counts (alerts last 24h/7d)
- inter-arrival time (gap between alerts)
- concurrent active regions count
- lag features (alert t-1, t-2…)

External enrich (optional, big value):
- frontline events / known strike dates
- weather (cloud cover affects drones)
- holidays / symbolic dates

# Tech Per Branch — Short

## 1. Classic stats (ARIMA/SARIMA, Prophet)

**Stack:** `statsmodels`, `prophet`, `pandas`, `matplotlib`.

**How works:**
- **ARIMA** = AutoRegressive + Integrated + MovingAverage. Predict next value from past values (AR) + past errors (MA). "Integrated" = differencing to kill trend. Needs stationary series.
- **SARIMA** = ARIMA + Seasonal terms. Catches weekly/daily cycles.
- **Prophet** (Meta) = decompose series into trend + seasonality + holidays. Fit curves, robust to gaps/outliers. Easy, less tuning.

**Use:** national daily alert counts → forecast next days.

---

## 2. ML tabular (XGBoost / Random Forest)

**Stack:** `xgboost`, `scikit-learn`, `pandas`.

**How works:**
- Reframe time series as supervised table. Each row = time window. Features = lags (count t-1, t-2), rolling means, hour, weekday.
- **Random Forest** = many decision trees, average vote. Reduce overfit.
- **XGBoost** = gradient boosting. Trees built sequentially, each fix prev tree errors. Strong on tabular.
- Bonus: feature importance → tells which signal predicts alerts.

**Use:** predict next-day alert count per region.

---

## 3. Deep TS (LSTM/GRU, Temporal Fusion Transformer)

**Stack:** `pytorch` or `tensorflow/keras`, `darts` (wraps many).

**How works:**
- **LSTM/GRU** = recurrent nets. Process sequence step by step, keep "memory" via gates. Learn long patterns past models miss.
- **TFT** = transformer for TS. Attention picks which past timesteps matter, handles many regions + external vars at once.
- Needs more data + GPU helps. Overkill for 2 days unless data rich.

**Use:** multi-region sequence forecast.

---

## 4. Anomaly detection (STL, Isolation Forest)

**Stack:** `statsmodels` (STL), `scikit-learn` (IsolationForest), `numpy`.

**How works:**
- **STL** = Seasonal-Trend decomposition by Loess. Split series into trend + season + residual. Big residual = anomaly.
- **Isolation Forest** = random trees isolate points. Anomalies isolate fast (fewer splits). Score by path length.
- Simple alt: z-score vs rolling mean. Spike if value > mean + 3·std.

**Use:** flag mass-attack days.

---

## 5. Spatial-temporal (Graph NN)

**Stack:** `pytorch-geometric` / `dgl`, `networkx`.

**How works:**
- Regions = graph nodes. Edges = geographic adjacency (drone flight path).
- **GNN** passes messages node→neighbor. Combine with temporal layer (GNN+LSTM) → predict spread A→B.
- Hardest. Skip unless core goal.

---
